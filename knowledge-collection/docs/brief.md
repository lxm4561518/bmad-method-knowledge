# 项目简报：内容提取与转录工具

## 执行摘要
这是一个基于 Python 的自动化工具，旨在通过将各种网络内容格式转换为文本来大幅提高知识获取效率。该工具接受一个 URL，自动识别内容类型（纯文本或视频），应用相应的身份验证（Cookie），并提取原文内容（不能包含任何修改或总结）——无论是抓取文本还是转录视频音频（把视频中的音频提取成文本），提升获取知识的效率。

## 问题陈述
### 当前状态与痛点
- **效率低下**：消费视频内容（如 Bilibili、抖音）非常耗时。一个典型的教育视频可能需要观看 60 多分钟。
- **格式锁定**：有价值的信息通常锁定在视频/音频格式中，难以略读、搜索或存档。
- **碎片化**：不同的平台（知乎、公众号、Bilibili）需要不同的访问方法和界面。

### 影响
- 用户花费过多时间过滤视频中的低密度信息。
- 难以从视频源中组织和检索知识。

## 提议的解决方案
一个统一的 Python 命令行实用程序（或脚本），作为通用的“内容转文本”转换器。

### 核心逻辑
1.  **输入**：用户提供 URL。
2.  **识别**：系统检测域名/平台（文本 vs 视频）。
3.  **验证**：系统检索特定平台的预配置 Cookie。
4.  **提取**：
    *   **A 类（文本）**：知乎、公众号、头条 -> 提取文章正文。
    *   **B 类（视频）**：Bilibili、抖音、腾讯视频 -> 下载音频/视频 -> 转录为文本 (ASR)。
5.  **输出**：包含内容的结构化（json格式输出，包含标题，原文内容，URL,采集当前时间）文本文件。

### 关键差异化优势
- **混合支持**：在一个流程中处理静态文本和动态视频源。
- **Cookie 感知**：使用用户提供的 Cookie 绕过基本访问限制。
- **专注于速度**：针对“获取文本时间”的减少进行了优化。

## 目标用户
### 主要细分市场：知识套利者与学习者
- **画像**：每天消费大量信息的人（研究人员、学生、内容创作者）。
- **目标**：以尽可能短的时间从内容中提取关键见解。
- **行为**：他们更喜欢阅读而不是观看；他们将知识归档在笔记中（Obsidian/Notion/Markdown/飞书 等等）。

## 技术要求（高层级）
- **语言**：Python
- **文本抓取**：BeautifulSoup / Newspaper3k / Selenium（如果需要）。
- **视频处理**：`yt-dlp` 或 Bilibili/抖音 的特定 API 包装器。
- **转录**：OpenAI Whisper（本地或 API）或类似的 ASR 服务。
- **验证**：Cookie 管理（可能通过 `browser_cookie3` 或手动字典注入）。

## 目标与成功指标
- **业务目标**：
    - 将内容消费时间减少 80%（例如，60 分钟视频 -> 10 分钟阅读）。
- **用户成功指标**：
    - 支持平台的提取成功率 > 90%。
    - 转录准确率 > 95%（取决于 ASR 模型）。
